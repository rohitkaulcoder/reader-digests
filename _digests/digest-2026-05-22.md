## Today's Top 3

### An Interview with Parallel Founder Parag Agarwal About Valuing Content on the Agentic Web ⚡
**Ben Thompson** · rss · 50 mins

1. Parag Agarwal (former Twitter CTO/CEO) founded Parallel in 2023 around a single obsession: AI agents will use the web 1000x more than humans ever have. Three years later he thinks that estimate was too conservative — the actual growth rate he's observing has already exceeded it.

2. Advertising works for the human web because it achieves near-infinite differential pricing: the top few percent of high-intent search queries and users subsidize the rest, capturing value proportional to each visitor's willingness to pay. Agents don't have attention and can't be distracted by ads, so this entire monetization mechanism collapses for agent traffic.

3. The three business models that dominate the web — subscriptions, advertising, and transactions — all depend on the attention economy. If human eyeballs stop arriving at a page, none of them generate revenue, yet the open web was built on the assumption that human traffic would always be there to pay for content creation.

4. Parallel is a search engine built specifically for AI agents, not humans — an API product that crawls, indexes, and searches the web with the output hyper-optimized for agent token efficiency and signal-to-noise ratio. Customers include Notion (when Notion Agent accesses the web, it goes through Parallel's APIs) and Harvey (which builds legal agents that spend $20–$50 per case doing deep web research).

5. Parallel's API is priced by use and allows buyers to dial their spend: a query can cost $0.10 or $2.40 depending on how much compute, retrieval depth, and premium content access the customer wants. The API even returns confidence scores so the calling agent can decide whether to spend more to resolve low-confidence sub-answers.

6. A large slice of what looks like consumer ad spend is actually B2B spend in disguise — a company like Rogo (which builds finance agents) advertising on Google to reach enterprise buyers is really doing B2B marketing with a consumer costume on. In an agentic future, those distribution dollars will flow through agent search rather than traditional search pages.

7. Shapley values, from Nobel Prize-winning game theorist Lloyd Shapley, solve the cooperative-game problem of fairly dividing a larger-than-sum-of-parts pie to maximize everyone's incentive to participate. The key intuition: if one seller has the only left glove and two sellers each have a right glove, Shapley math yields ~4× more compensation for the left glove — scarcity and uniqueness are directly priced in.

8. Computing exact Shapley values requires running every counterfactual (remove each content source and re-evaluate the agent's answer quality), which for 100,000 sources costs more compute than the value being divided. Parallel spent a year building a model that estimates Shapley values — they first ran actual expensive simulations to generate training and eval data, then trained the estimator against those ground-truth outputs.

9. The Index by Parallel (launched this week) exposes four metrics for any website: Impressions (how often agents surfaced the content during real work), Citations (how often agents actually grounded answers in it), Value (whether the content appeared in high-stakes expensive queries, like a banker's $20 deep-research run versus a cheap casual query), and Uniqueness (the Shapley score — does removing your content meaningfully degrade the answer?).

10. nyc.gov scores exceptionally high on Uniqueness because it holds arcane, authoritative, non-duplicated facts about New York law that no other source replicates — a perfect Shapley-high asset. Stratechery, by contrast, scored well on Value but dropped on Uniqueness; Agarwal's explanation is that it's built for human reading rather than agent fact retrieval.

11. PR Newswire is one of Parallel's launch partners and scores disproportionately well because it holds authoritative, timestamped factual records of who said what — the kind of content agents cite when grounding claims. The content is open-web (not paywalled) but Parallel is paying for it anyway, on the theory that the open web will disappear if no payment mechanism exists.

12. Paywalled content partners at launch include PitchBook, Tracxn (business intelligence data), Fortune, and The Atlantic. The Index system treats both open-web and paywalled content uniformly — both get valued by the Shapley estimator, and both receive payouts proportional to their measured contribution to agent work.

13. Local news has unexpectedly high Uniqueness scores because on-the-ground local facts have no substitutes. This quantitatively validates what was previously a qualitative argument: unique factual reporting about specific places is more valuable to agents than generalist analysis, because three generic sources all covering the same national story have near-zero marginal Shapley contribution each.

14. Agents also disproportionately reward "recipe" content — articles that lay out a plan or methodology for solving a class of problem (e.g., a structured guide to financial due diligence). Agents read these, verify the plan is sane, cite it, and follow it — so procedural how-to content earns unexpectedly high Citations scores alongside factual content.

15. Parallel's system distinguishes inference-time data (content agents access while doing work) from training/pre-training data. Training data attribution is currently unsolvable — there's no way to cite which text influenced a model weight — so Parallel explicitly built only for inference-time use, where citations are explicit and Shapley calculations are tractable.

16. Content for humans and content for agents are likely to diverge into separate ecosystems. Agents don't care about narrative quality, storytelling arc, or writing style — they extract facts and methodology. Humans increasingly value unique shared experiences and community around content. Some content (like high-quality factual reporting) may serve both, but the optimization targets are different.

17. Cloudflare is attempting the supply-side version of this problem: creating scarcity on behalf of content owners by controlling crawler access. Agarwal agrees with Cloudflare CEO Matthew Prince's diagnosis (the open web is at risk), but Parallel approaches from the demand side — knowing what content is actually worth to agents — which gives content owners a high-confidence BATNA in any direct deal with a model lab.

18. OpenAI and Anthropic's bundled search tools within their APIs are the default competitor, but they're built around human search ergonomics. Parallel's argument is that this is a sufficiently specialized tool — token-efficient, agent-native, with an economic layer for content owners — that it justifies a dedicated product. Customers migrate to Parallel as they mature beyond initial prototypes and optimize agent pipelines.

19. Parallel's business is an adjacency to model providers, not a competitor: customers buy Parallel APIs alongside model tokens. When OpenAI's API usage grows at "crazy rates year-on-year," Parallel sees the same growth. At the time Agarwal started the company 2.5 years ago he felt he "was crazy" predicting 1000x more agent web use — today he thinks even that was an underestimate.

20. The supply-side virtuous cycle has already started: adding the initial launch partners (PR Newswire, PitchBook, Fortune, The Atlantic, independent Substack writers like Azeem Azhar and Alex Heath) materially improved Parallel's answer quality for demand-side customers, demonstrating the feedback loop without needing to wait for a massive content catalog.

21. Agarwal's vision for content-creator incentives mirrors how social networks created UGC supply — but instead of status and attention as the motivator, it's direct payment for filling factual gaps agents need. Anyone who identifies an underserved niche, creates a website with authoritative unique content, and registers with the Index can extract disproportionate Shapley-value compensation relative to their effort.

22. Twitter's historical resistance to machine learning in its consumer product was an identity-based mistake — the company defined its reverse-chronological feed as a differentiator against Facebook and conflated "no algorithm" with "better recency." Agarwal pushed ML into the consumer product after his ads-side data showed Twitter DAU was set to saturate at ~90 million; after ML adoption, DAU grew consistently until he left.

23. As Twitter CTO, Agarwal built the ads ML team from scratch (starting at ~10 people building ads, scaling to revenue growing from near-zero to several billion dollars over a few years) and scaled the underlying infrastructure from 4 machines to 4,000 machines solving the same problem. He takes accountability alongside Jack Dorsey for Twitter never becoming a better business during those years — specifically failing to push harder on ad relevance, performance ads, and operational efficiency.

24. Agarwal views the mass Twitter layoffs under Musk as unsurprising: the engineering was well-built and the team was talented, but the organization had excess headcount and was slow at product innovation. The product not crashing was an expected outcome given the underlying system quality, not a surprise.

25. Agarwal grew up in the Bhabha Atomic Research Centre colony in Bombay — a 2×2 mile government compound where every family had a parent working on nuclear research. He credits this peer group, not wealth or resources, for his competitive intellectual formation; he didn't interact with a computer until age 17 and didn't fly on a plane until the Physics Olympiad, where he won a gold medal in 2001.

**Quotable:**
> "If we show up with this mindset that the open web is out there so I don't need to pay for it, it's not going to be out there — and only large businesses that have private deals will have access to open web content, and I may not have access." — *Agarwal explaining why Parallel is paying content owners even when it could legally scrape them for free*

### You'll lose your job in 2027. ⚡
**Lenny's Newsletter** · email · 8 mins

1. **Your current role is already expiring, not just at risk.** Intuit and Meta have already announced cuts in 2025, and even employees who keep their seats will find their responsibilities, tools, scope, and headcount expectations rewritten. Elena Verna describes "firing herself" out of parts of her own role — some tasks became irrelevant, some moved to others, some were automated — while technically staying at the same company with the same title.

2. **Full-time employment was never as stable as it felt — and now it's actively precarious.** Junior employees are the first to be cut; senior ones are expensive line items. The safety net of a traditional job was always fragile, and AI-driven transformation has simply made the timeline visible. Treating a full-time role as stable now is, in Verna's framing, "staying on the Titanic while the lifeboats float away."

3. **Solopreneurship is the lifeboat, and it takes time to build — so start today.** Verna has spent five years building consulting, advising, and freelance income that now exceeds her full-time cash comp. The path requires acquiring clients (LinkedIn, Substack, counter-pitching recruiters), defining your unique value proposition, and moving from hourly rates to monthly retainers. Because it takes months to build momentum, the time to start is before you're forced to.

4. **"Mom 'n' Pop SaaS" is a viable second path: small, profitable, and yours.** AI tools have eliminated the need for a technical background to ship software. A niche product generating $3k/month in semi-passive income is the goal — not venture-scale. The economics now favor this because development costs are collapsing.

5. **AI-native is a six-tier skill ladder, not a binary switch.** Tier 1 is using AI to sharpen written communication. Tier 2 is a meeting partner (Verna uses Granola for todos, commitments, and manager feedback). Tier 3 treats AI as a thinking partner that challenges logic rather than flatters. Tier 4 builds prototypes and internal tools. Tier 5 ships code to production — even for non-technical people. Tier 6 deploys agents for repetitive, structured tasks, though agents require significant setup and oversight and are often overhyped. The most critical meta-skill across all tiers: creating clarity, because "AI just helps you do the wrong thing faster" if you don't know what matters.

### The New Escape Velocity ⚡
**Byrne @ The Diff** · email · 9 mins

1. Companies hit "escape velocity" when an ad-hoc process gets systematized into metrics — at which point management can treat it as an abstraction and redirect attention to the next unsolved problem. The canonical example is consumer growth: Zuckerberg inviting roommates, Snapchat's Evan Spiegel recruiting near sororities, Reddit's founders running sock-puppet accounts to seed content until real users took over. Steve Huffman's description of Reddit reaching that moment — "I remember the first day, a few months in, when we didn't have to submit any content" — is the cleanest definition of the concept.

2. At sufficient scale, product decisions stop being creative judgments and become statistical ones: button placement and CTA wording get A/B tested rather than debated. This has a combinatoric limit — the losing variant from test A combined with the losing variant from test B can outperform the isolated winner of each — so large changes still require modeling and judgment, but small, isolated feature changes become purely empirical.

3. Scale also surfaces every UX failure in raw logs: how many users abandon nested menus and fall back to search, how many user-hours are wasted filling out forms that dead-end in errors. Pre-AI, the natural endpoint was an automated pipeline that statistically identifies friction points, ranks them by business impact, and produces a ranked to-do list for engineers.

4. The new AI-era escape velocity goes further: an internal system that autonomously identifies friction points, generates a fix, tests it on a subset of users, and ships it to production if the subset responds well — targeting specifically the new users who resemble the test cohort. This closes the loop without requiring a human at each step.

5. The downside is real and not accidental: several high-profile companies reported major outages shortly after publicizing their vibecoding practices. The analogy is early Microsoft shipping hand-optimized assembly code, then shipping bloated software as high-level languages reduced human thought per instruction — the tradeoff is inherent. Some companies will be killed by pushing slop to prod; more will go into terminal decline because they refuse to.

6. The strategic implication inverts conventional product wisdom: a broad-but-mediocre product with a massive marketing budget beats a focused excellent one, because mediocrity generates more failure data to iterate on. If you never mess up, you never collect information about how to improve. When optimization is cheap, the optimal move is to ship maximum surface area and perfect it later.

7. **(News: Anthropic)** Anthropic revenue rose 130% Q/Q to $10.9bn with a $559m operating profit — which is framed as an embarrassing capital-allocation failure, not a success. Earning a profit means Anthropic underestimated demand for high-margin inference and failed to deploy enough infrastructure. The gap is being papered over by buying $15bn/year of compute from xAI, neatly offsetting Elon Musk's own capacity miscalculations. Dario Amodei, asked whether he'd prefer the profit or a loss at this revenue level, should "ruefully concede he should have listened to Dwarkesh."

8. **(News: OpenAI lock-in)** OpenAI is offering to invest token credits in every current YC batch startup in exchange for equity — functioning simultaneously as a lock-in mechanism (switching costs once you're embedded in one model's quirks), a credibility signal to enterprise CIOs, and a potential new financial instrument Sam Altman predicted would be needed. OpenAI is also experimenting with 1–3 year "compute bonds" where customers prepay cash for future inference access, meaning its S-1 will carry an unusually complex balance sheet.

9. **(News: IPO race)** OpenAI is preparing to file its S-1, potentially going public in September. Beyond the capital raise, there is a finite pool of institutional capital that can absorb companies the scale of SpaceX, OpenAI, and Anthropic simultaneously. Going public first makes it harder for the next entrant — one of the most valuable things OpenAI can do right now is keep Anthropic slightly short of capital.

**Quotable:**
> "There probably will be companies that are killed specifically because they pushed slop to prod. But there will also be plenty of companies that go into terminal decline because they won't." — *on the stakes of autonomous AI-driven product iteration*

## AI Economics & Policy

### Free Lunch: Tax revenues in an AI world 📌
**Delphine Strauss** · email · 5 mins

1. **Labour taxes — the fiscal bedrock most at risk from AI — represent nearly half of OECD revenues.** Personal income tax accounts for roughly one-quarter of total tax receipts across OECD countries, and social security contributions (paid by both employers and employees) account for another similar share. The US is even more exposed, relying more heavily on labour taxation due to the absence of a national consumption tax. Tom Blomfield, Monzo's founder, predicted last month that income tax could be "obsolete within five years," arguing future governments will tax compute rather than human labour.

2. **Windfall Trust modelling suggests the net fiscal hit may be near-neutral — under optimistic assumptions.** New research modelled an "average" OECD economy under a "low impact" scenario where 12% of workers are displaced (translators, junior developers, paralegals) but AI productivity gains stay largely with domestic companies and consumers. Co-author Deric Cheng called the outcome "a little less disruptive than we might have expected." The risk scenario — where large monopoly profits flow offshore to US tech giants — is where fiscal exposure becomes severe, and Trump's rejection of the OECD global minimum corporate tax deal illustrates how hard international coordination would be.

3. **Existing tax tools — consumption taxes, equalised capital-gains rates, land and inheritance levies — may be sufficient if AI displaces high-paid white-collar workers rather than low earners.** Warwick's Centre for the Analysis of Taxation director Arun Advani argues that if AI makes goods cheaper, a consumption tax "could do a lot" of heavy lifting, and its usual regressive sting would be blunted if AI erodes wage differentials by hitting higher-paid workers hardest. Novel proposals — OpenAI's call for higher capital-gains and corporate taxes, Bernie Sanders' "robot tax," Anthropic CEO Dario Amodei's "token tax" on AI model outputs — all carry distortion risks and could push companies to lower-tax jurisdictions, but Advani warns the risk of a "crazy giant shock" means governments need to prepare these options fast.

**Quotable:**
> "I don't think we'll tax human labour, we'll tax compute." — *Tom Blomfield, Monzo founder, on income tax potentially being obsolete within five years*

## Developer Tools & Infrastructure

### What is Next.js? 📎
**Technically** · email · 8 mins

1. When you type a URL into a browser, six steps happen — DNS lookup, IP resolution, server connection, HTTP request, server response, page render. Web frameworks handle steps 3–5: routing requests to the right code, serving the right page, and gracefully handling failures. Without a framework, a developer must manually wire up every URL, every HTTP method, and every edge case from scratch.

2. Web frameworks are language-specific and fall on a spectrum from opinionated to flexible. Django (Python) imposes strict file locations, naming conventions, and URL configuration files — but rewards compliance with a free admin panel and ORM. Flask (also Python) requires manually defining every URL and HTTP method, giving more flexibility at the cost of more setup. Neither approach is universally better; the right choice depends on the project and the developer.

3. Next.js is an opinionated JavaScript framework that layers on top of React specifically. Its file-system routing is the clearest example: drop a file called `sponsorships.js` into the `/pages` folder and Next.js automatically serves it at `/sponsorships` — no manual routing config required. Its performance features go further: automatic link preloading, a choice between server-side and client-side rendering per component, and built-in caching so pages don't re-run requests on every load.

4. Next.js is free and open source but owned and maintained by Vercel, a for-profit company — an arrangement with no real parallel among major web frameworks except Gatsby. Critics argue this creates a conflict of interest that manifests in undocumented behaviors, opaque roadmap decisions, and a weaker community OSS ecosystem.
